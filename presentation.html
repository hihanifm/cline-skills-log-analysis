Cline Skills Project

Turning Tribal Knowledge
into AI-Powered Tools

A modular skills system that lets AI assistants diagnose Android connectivity issues from raw logs and network captures — the same way an expert engineer would.

Android Diagnostics Cline / VS Code Log Analysis PCAP Analysis Two-Stage Pipeline
The Problem

Diagnosing Connectivity Issues Is Hard

🧠

Expert Knowledge, Not Written Down

Knowing which logcat tags matter for an IMS registration failure lives in an engineer's head — not in any document.

📂

Massive, Noisy Files

Android dumpstates are hundreds of megabytes. PCAP captures contain thousands of frames. Finding the signal in the noise takes hours.

🔄

Repeated Every Time

Every engineer on every new case manually greps the same patterns, decodes the same SIP messages, and writes the same summaries.

The Solution

A Two-Stage Analysis Pipeline

Stage 1

Context Builder

Deterministic. Runs ripgrep / tshark filters defined in YAML templates. Outputs a structured context.txt.

→
Stage 2

LLM Synthesizer

Reads context.txt and generates per-pattern summaries + a final diagnostic report using Cline or Anthropic API.

→
Output

Human-Readable Report

report.md with findings and summary.md with the final diagnostic — ready to share.

Stage 1 is fully deterministic and reproducible. Stage 2 is the only place an LLM is involved.

Architecture

Modular Skill System

workflow-orchestrator Cline-facing entry point. Runs the full pipeline end-to-end given a workflow file and an input path.
template-engine Executes YAML filter definitions against log/PCAP files using ripgrep or tshark. Auto-detects the right tool.
template-library Stores reusable filter YAMLs for common patterns (wakelock, IMS SIP, RIL, DNS, HTTP). Browse with the library skill.
log/pcap-template-creator Guided authoring of new filter templates — asks for patterns, tests against sample files, writes the YAML.
workflow-creator Guides creation of new end-to-end workflows combining multiple templates with a final summary prompt.
Technology

Built on Proven Tools

Log Filtering

ripgrep

Regex-based filtering of Android logcat and dumpstate files. Handles binary/null-byte files with --text. Context-aware line capping.

Network Analysis

tshark

Field extraction from .pcap / .pcapng captures. Supports SIP, DNS, HTTP display filters with structured column output.

Orchestration

Python + YAML

Pure stdlib Python scripts. YAML workflow definitions with frontmatter. Single PyYAML dependency. Runs anywhere Python 3 runs.

AI Integration

Cline / Anthropic API

Two backends: Cline fills SUMMARY_PROMPT markers directly in VS Code, or Anthropic API is called for fully automated reports.

What's Next

Roadmap

1

Expand the Template Library

Add more filter templates for WiFi, Bluetooth, thermal, ANR/crash patterns. Build a richer library that covers more common issue categories.

2

Post-Processor Ecosystem

More decode scripts (decode_sip.py is a start). AT commands, hex dumps, RIL message decoders — make raw output human-readable before it hits the LLM.

3

Team Sharing

Project-local filters/ and workflows/ committed to repos. Teams build shared diagnostic playbooks alongside their code.

4

Automated CI Integration

Run workflows automatically on test farm failures. Surface diagnostic summaries in PRs before a human even looks at a log.

Codifying Tribal Knowledge

The best engineers carry years of pattern recognition in their heads. This system makes that knowledge reusable, shareable, and available to anyone on the team — including an AI.
